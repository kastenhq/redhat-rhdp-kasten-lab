üìñ Part 1. Configure cluster storage for Kasten

{: .important } > Prior to configuring backups, we need to ensure our cluster storage is capable of performing snapshots of our persistent data workloads. > Because our cluster is deployed on top of Amazon EC2, we can leverage EBS storage and it‚Äôs associated CSI driver to perform volume snapshot operations. > > To do so, we need to ensure our VolumeSnapshotClass object created on the cluster for the ebs.csi.aws.com provisioner is correctly annotated. We need to add the annotation > k10.kasten.io/is-snapshot-class: "`true`"

  1. On the bastion host, run the following command to patch the csi-aws-vsc VolumeSnapshotClass so Kasten can use it to perform snapshot operations

    oc annotate volumesnapshotclass $(oc get volumesnapshotclass \
-o=jsonpath='{.items[?(@.metadata.annotations.snapshot\.storage\.kubernetes\.io\/is-default-class=="true")].metadata.name}') \
k10.kasten.io/is-snapshot-class=true

  2. Run the following command to validate the VolumeSnapshotClass has been annotated

    oc get volumesnapshotclasses.snapshot.storage.k8s.io csi-aws-vsc -o yaml

üìñ Part 3. Create A Policy

Before performing a Kasten install, the “Primer Script” can quickly spot any potential problems with our cluster configuration.

  1. Click < Dashboard to return to the Kasten main dashboard, click Applications to view all applications discovered by Kasten on the cluster.

    Figure 5. click apps

  2. Type pacman in the filter search bar, then click the Create a Policy button.

    Figure 6. create policy button

  3. Leave the default name and optionally add a comment. By default, the policy will generate hourly snapshots of the application with a standard retention policy. Leave these defaults.

  4. Scroll down slightly and toggle Enable Backups via Snapshot Exports to the ON position and select minio as the Export Location Profile.

    Figure 7. enable exports

  5. Under Select Applications observe that we are explicitly protecting all resources in the pacman namespace.

    {: .note } A single Kasten policy can also protect multiple namespaces, and can even do so dynamically via Kubernetes labels.

  6. Leave all other defaults and click Create Policy.

    Figure 8. create policy

  7. You will be returned to the Policies page. On your new pacman-backup Policy, click the YAML button to view the Kubernetes manifest.

    Figure 9. policy yaml

    As native Kubernetes resources, Kasten policies, profiles, and even running backups or restores can be easily implemented via oc or API. Exposing the YAML through the UI makes it easy for administrators to copy and modify existing examples of Kasten resources or actions.

  8. Click Cancel to close the YAML window.

üèÅ Part 4. Takeaways

  • Local snapshots are not backups

  • Configuring off-cluster storage for backup is a good idea

  • Configuring an S3-compatible bucket in Kasten can be done in a few clicks

  • Because Kasten is K8s-native, all actions, policies, resources can be implemented using oc commands or API
